The grouping does double duty: column "backed by" is the applicability matrix in data — a type's routable set is "every concept whose backers include it." Shared concepts (A) get one paramKey, so a route survives a track changing type. Range is a class, not a number — the registry holds the concrete bound; the route stores depth % only. Some PhaseFlux/Curve concepts show uni/bip because polarity needs your confirm (e.g. PitchWarp ± vs 0→full). Keep/drop of the non-now rows is still your triage.
