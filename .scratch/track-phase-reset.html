XFORMER tracks — the 0 → 1 phase & resetMeasure

Every grid track derives a phase that wraps 0 → 1 over its cycle. resetMeasure hard-realigns that phase to the bar (relativeTick = tick % resetDivisor; on 0 the engine calls reset()). Below: how each track shapes the 0→1 ramp, and what its reset clears. MIDI/CV and Teletype are excluded — they aren't step-phased.

Step position from a ramp or an external CV scan; stages fire on threshold crossings.

PhaseFlux

4×4 cells along a cycle; per-stage curves sampled into CV each PPQN tick.

Note on resetMeasure: it is sequence-owned per track, so tracks can wrap on different bar counts — set them unequal and the tracks phase against each other, realigning only where their reset periods coincide.
