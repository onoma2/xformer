PhaseFlux — distributing the 0→1 cycle across 16 stages

PhaseFlux walks 16 cells (a 4×4 grid, in traversalPattern order). The cycle phase runs 0→1 once per cycle; each stage owns a slice of that 0→1 whose width is its own — stages are not evenly spaced. CycleLength (Adaptive / Fixed) decides what a skipped stage does to the partition.

How a stage's slice width is computed

slotWidth[cell] = stageDivisorTicks(stageDivisor)        // base slot at the 1/16 reference
                × (stageLen + lenNudge) / 64             // per-stage length: 64 = ×1, 0 = ×0, 127 ≈ ×2
                × seqDivisor / referenceDivisor          // global divisor scales the whole cycle
                × 100 / clockMultiplier                  // clock mult: 50 = ×2 longer, 150 = ×⅔

cumulative[i+1] = cumulative[i] + slotWidth[ traversal[i] ]
cycleTicks      = cumulative[16]
cyclePhase      = relativeTick / cycleTicks              // 0 → 1 over the whole cycle
activeCell      = bin(cyclePhase) via cumulative[]        // which slice we're in
stagePhase      = (pos − cumulative[i]) / slotWidth[i]    // 0 → 1 within that stage

Floor: if the whole cycle comes out shorter than measureDivisor/32, every endpoint is proportionally stretched — relative weights preserved, no cell ever snaps to zero length.

The partition — one 0→1 cycle, 16 uneven slices

What sets each slice

CycleLength — what a skipped stage does to the cycle

Adaptive

A skipped stage gets slotWidth = 0 — it drops out. The cycle is the sum of only the active stages, so it shrinks and wraps sooner. Active stages keep their absolute width; the 0→1 just spans fewer of them.

Fixed

A skipped stage keeps its slot — the engine emits silence during it. The cycle length is unchanged regardless of skips, so active stages stay at the same absolute position; the 0→1 still walks through the silent slot.

Order vs width: traversalPattern sets which cell occupies each slot (the walk order through the 4×4 grid); stageDivisor/stageLen set how wide each slot is. Together they place all 16 stages along the single 0→1 cycle, and CycleLength decides whether skips compress that cycle (Adaptive) or leave gaps in it (Fixed).
