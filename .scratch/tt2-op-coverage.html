Teletype ops — upstream vs TT2 native table

Branch feat/tt2-v2-native, current state (2026-06-15). The computational/musical core is complete — every family below is wired in the native op table and covered by a passing TestTeletypeV2* suite. Performer-native bridges (W* / BUS / RT / MO.* / G.*) replace upstream's i2c hardware ops. Only i2c hardware-module families (ansible, telex, grid, just friends, WW, disting) and multi-scene are out of scope by design.

Control-flow language is complete — IF / ELIF / ELSE, PROB, EVERY, SKIP, L (loop), the DEL family (DEL / .X / .R / .G / .B), and the ms metro are handled as pre-op modifiers in lowering + the runner. Their op-side hooks (BREAK/BRK, KILL, SCRIPT/$, DEL.CLR, M/M.ACT) are wired in the table.

Out of scope

Upstream i2c hardware-module ops are intentionally not parsed or wired — they target external Eurorack modules TT2 does not bridge: ansible, telex (TELEXi/TELEXo), grid/monome, just friends (JF.*), white whale (WW.*), disting EX. Multi-scene (SCENE/PRESET) is also out of scope (Performer uses its own project/pattern model).
Source: native op table in src/apps/sequencer/engine/TeletypeNativeOps.cpp (handler registrations) vs teletype/src/ops/*.c (upstream). Family status reflects table coverage + passing TestTeletypeV2* suites as of branch feat/tt2-v2-native.
