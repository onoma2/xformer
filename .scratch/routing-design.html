Routing Mod-Matrix — the design in one read

What the overhaul actually decides, and why.

One idea drives everything: sources are few, shared, scarce (4 CV in, 8 modulators); targets are many, per-track (10–30 params × 8 tracks). The current flat enum forces both through one list. The design splits the system along that asymmetry — broadcast the scarce things globally, manage the abundant things locally — and makes the engine ignorant of meaning so heterogeneity lives in tables, not a smeared enum.

1 · The engine is name-blind

A route delivers a normalized number into an address. The engine never knows the target's name, range, or meaning:

scope(t).applyRouted(slot.paramKey, v) // no name, no range, no switch
address
(scope, paramKey) — replaces the flat 103-entry Target enum.
meaning
a per-scope param table owns name + range + applyRouted(). One row per routable param.
applicability
= table membership. A Tuesday track has no FirstStep row, so it can't be picked or silently no-op.
paramKey
append-only numeric key, never the array index — so adding a row never shifts saved routes.

2 · Two surfaces, split by broadcastability

① Global · masked pool

Scarce source → many tracks

~8 fat slots. Keeps the track-mask, so one CV → all tracks' Transpose is one route, one linked edit. Per-track depth + shaper live here (the ×8 RAM, contained).

  • Tier 0 — Performer-wide (Tempo, Play, Swing)
  • Tier 1 — universal (Mute, Run, Divisor, OutputRotate)
  • Tier 2 — pitched (Scale, Transpose, Octave) + Phase
② Per-track · local matrix

The per-track explosion, in context

N≈3–4 thin slots per track, single-track (no mask, no ×8). Generalizes Indexed's RouteConfig; assign-from-param (long-press a param → local route).

  • Tier 3 — step-range (Note/Curve/Indexed only)
  • Tier 4 — type-specific (Tuesday, Stochastic, PhaseFlux stages…)
  • cell / group scope · the track's own inlets

3 · Three kinds of target

direct
writes a param straight (Octave, Tempo, Stochastic Complexity).
inlet
fills a named per-track CV bus (_routedX) — the global→local bridge. Indexed A/B, DiscreteMap Input/Scanner. Each type owns its own; no borrowing.
rail
the Bus (BusCv1–4) — Engine-owned shared CV, both source and target.
global: source ──route──▶ track inlet (_routedX) local: inlet ──route──▶ direct param / cell-group (Indexed & DiscreteMap already do this)
