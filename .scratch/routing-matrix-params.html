The structure idea (refined). A band earns a fixed cross-engine tab only if many engines own it — so it's dense across the 8 columns. That's really just Pitch, Clock, and SlideTime. The old "Prob" tab was fake: the biases are Note-only / Curve-only → they're engine params (now folded into Note's and Curve's Engine tab). Same goes for the Step/Move set (RunMode/First/Last/Rotate/Offset) — sequencer-family, owned by 2-3 engines, not universal.

Everything engine-flavored lives in one dynamic Engine tab = whatever engine the cursor's track is (Stochastic → 15 rows, Tuesday → 9, Note → its 4 biases). Its rows come straight from the focused track's ParamTable minus the shared Pitch/Clock keys, so it's always complete.

Result: ~4 fixed tabs (Pitch · Clock · SlideTime · Global) + the one context Engine tab cover all 65 params, not 20 pages.

Open: does SlideTime ride with Clock (→ 3 fixed tabs)? Step-family (RunMode/First/Last/Rotate) — its own "Step" tab for the sequencer engines, or just in each Engine tab? Engine tab follows cursor vs. held? Base-0 inlets (DMap/Indexed) — Engine tab, or their own row-kind (they're sinks, no base)?
