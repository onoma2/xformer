Engine — name-agnostic apply (single-pass per-tick recompute)

for each active slot: v = source(slot.source) v = shape(slot.shaper[t], v, state[t]) · per-track shaper (global pool) / scalar (local) v = depthWindow(min,max,v) · depthPct[t]/biasPct[t] v *= scaleValue(slot.scaleSource, t) · from VcaNext, explicit now scope(t).applyRouted(slot.paramKey, v) ← NO target name/range/switch in engine
Each scope owns a param table = the single source of truth: ParamRow { name, range, flags(direct|inlet|fixed|continuous|discrete|structural), applyRouted() }. Adding a routable param = adding one row. paramKey = append-only numeric key (never the array index).

The inlet bridge (global → local, two engines already do this)

global page: source ──route──▶ track INLET (_routedX) e.g. Indexed A/B, DiscreteMap Input/Scanner/Sync │ local matrix: INLET (or any src) ──route──▶ direct param / cell-group · Indexed: A/B → RouteConfig → cell Duration/GateLength/NoteIndex (scale-aware) · DiscreteMap: Input/Scanner → hardwired engine consumption
Three target kinds: direct writes a param · inlet fills a named per-track CV bus (_routedX), each type owns its own (no borrowing) · rail (Bus) Engine-owned shared CV, src+target.
