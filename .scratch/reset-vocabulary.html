The many meanings of “reset”

One word, at least five genuinely different operations across the model, transport, and per-track layers — and reset() itself clears a different set of state in every track engine.

The five families

OperationLayerWhat it actually doesTriggered byWeight
clear() / init Model Wipes a sequence/track to factory defaults. Content is gone — this is the “reset to nothing” you meant. New project, INIT action, clear command wipe
Engine::reset() Transport · global Cold-resets every track engine + MIDI-out + modulators. The big transport “back to top.” Boot, Clock::Reset event (MIDI Start, transport reset) cold · all
Engine::clockReset() Transport · clock Zeroes the master clock tick/phase + routing shaper state. Does not touch sequence content or cursors. Shift+Start while running, clock-input Reset mode clock phase
TrackEngine::reset() Per-track COLD. Cursor→start, clears CV/gate, output queues, accumulators, seek state, record history, then changePattern(). Engine::reset(); the routed Reset target cold
TrackEngine::restart() Per-track WARM. Cursor→first step only. Keeps CV/gate, queues, accumulators; no changePattern(). Song-slot advance (song mode) warm
resetMeasure Model · config A setting, not an action: “auto re-anchor every N bars.” A schedule that fires a restart. User sets it per sequence config
External Sync Per-track · DM/Indexed Re-anchor first-tick on an external CV edge. Today calls the COLD path (resetSequenceState()). Routed sync CV (rising edge) cold should be warm

wipe destroys content cold clears runtime + output warm reposition, keep continuity clock tempo phase only config a setting, not an act

Also in the family, smaller: SequenceState::reset() (run-mode cursor only), Accumulator::reset() (back to minValue), tapTempoReset() (averaging buffer — unrelated to playback), resetSequenceState() (the Indexed/DiscreteMap inner reset-to-first-step shared by reset() and External sync).

“Full reset” is a different thing per track

Same call, reset() — but the set of state it clears is engine-specific. This is why “reset the track” can mean a thousand things:

TrackWhat its reset() actually clears (beyond cursor→start)
NoteRe:Rene seek X/Y + divisors, accumulators, record history, gate/CV queues, pulse counter, prev-condition.
IndexedRe-rolls the runMode advance (RNG picks the next step), step/gate timers, slide, steps-remaining, free-tick anchor.
DiscreteMapInternal ramp phase → −5V, prev-input crossing, pluck envelope, re-samples octave/transpose/rootNote, threshold cache.
Curve / Tuesday / Stochastic / PhaseFluxEach its own phase/shape/seed/playhead state — no shared definition of “cleared.”

The musical mismatch you're pointing at

Drift-sync wants WARM, but External Sync wired COLD

The drift-correction “re-anchor the first tick” you described is restart() semantics — reposition the playhead, keep CV/queues/accumulators flowing. But External Sync today reaches for the cold resetSequenceState(), which dumps output and runtime state — an audible jolt, especially on the continuous tracks (Curve, PhaseFlux, DiscreteMap).

So “universalize Sync” isn't just porting the branch to all tracks — it's binding it to warm re-anchor, not cold reset.

A vocabulary that stops overloading “reset”

Five concepts, five names — so a label never has to mean a thousand things:

Clear / Init
Wipe content to defaults (model).
Clock Reset
Zero the master clock phase (tempo, not notes).
Reset
Cold per-track / global — back to top, clears output. (Transport & the routing Reset target.)
Restart / Rewind
Warm reposition — cursor to first step, keep continuity.
Sync / Re-anchor
External first-tick realign for drift — warm, per-track, universal (the feature to build).

Where this lands for routing-matrix

Sync becomes a universal warm re-anchor inlet (sibling of resetMeasure), distinct from the cold Reset target. That dissolves the Indexed→DiscreteMapSync borrow by promotion (everyone owns Sync), and keeps Input/Scanner as DiscreteMap's genuinely-unique inlets. Pairs with the drift work in wallclock-time-architecture.
