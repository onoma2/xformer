Routing → Mod-Matrix Behavioral Structure

Research artifact, 2026-05-31. No direction chosen. Goal: pick a clean behavioral model for routing and separate concerns at both the UI/behavior level and the engine level. Companion to .tasks/core-architecture-optimization/routing-five-sources-map.md.

1 · Mod-matrix paradigms (what the field does)

Two independent axes. Performer today sits in the worst corner of both: one flat global table (engine), one central list page (UI).

Axis A — Engine: where routing lives

ModelWho owns a routeExamples
Global flat matrixone central table, all sources × all destinationsPolyBrute, classic synths, Performer now
Per-scope / per-devicerouting lives where the target parameter lives; local source/target vocabularyBitwig "unified modulation", Performer's own Indexed RouteConfig

Axis B — UI: how a route is made

ModelInteractionExamples
Matrix / list pagecentral page, rows=src cols=dest, edit the cellPolyBrute grid, Performer RoutingPage
Hold source → desthold a source, tap a destination, depth pops upHydrasynth (32 conns)
Assign-from-parameterlong-press the param you want moved → pick a sourceBitwig, Vital, Logic Sampler

Common matrix-slot anatomy across all of them: source → destination · depth (bipolar/unipolar), fixed N slots ("virtual patch cables"). Performer's Indexed RouteConfig already is a slot: { source, targetParam, amount −200..+200%, targetGroups } × 2 slots + combine mode.

2 · Current Performer routing — the smear

One flat Routing::Target enum (103 entries) × Routing::Source (37). A single routable target's identity is spread across five places that must move in lockstep; miss one → silent dead route:

  1. Target enum + First/Last sentinels — model/Routing.h
  2. targetInfos[] metadata table — Routing.cpp:375
  3. writeTarget dispatch switch (per trackMode) — Routing.cpp:208
  4. picker applicability + isPerTrackTarget — RoutingPage.cpp
  5. per-model writeRouted leaf — each *Track/*Sequence

Live drift bugs this produces: StochasticFeel routes but never writes (no leaf case); PhaseFlux per-stage params never added anywhere; Indexed dual-sources its targets; Stochastic Divisor/Scale/RootNote special-cased because its divisor isn't Routable. Built for 2 homogeneous types (Note/Curve), now stretched over 8 heterogeneous ones.

3 · Current routes, bucketed

The enum already groups by category. Re-cast into the three buckets you asked for. Sources feed all buckets: CvIn1–4, Midi, Mod1–8 (global modulators), CvOut1–8 / GateOut1–8 / BusCv1–4 (output-feedback).

Whole-Performer

global, one instance, no track index

  • Engine: Play, PlayToggle, Record, RecordToggle, TapTempo
  • Project: Tempo, Swing, CvRouteScan, CvRouteRoute
  • Bus: BusCv1–4 (internal patchbay — also sources)

Per-track, generic

every track type, indexed by track

  • PlayState: Mute, Fill, FillAmount, Pattern
  • Track: Run, Reset, SlideTime, Octave, Transpose, Offset, Rotate, 5× ProbabilityBias, Cv/GateOutputRotate
  • Sequence: FirstStep, LastStep, RunMode, Divisor, ClockMult, Scale, RootNote

Per-track-TYPE

only meaningful for one engine

  • Tuesday: Algorithm, Flow, Ornament, Power, Glide, Trill, StepTrill, GateOffset, GateLength
  • Curve: Chaos×4, Wavefolder Fold/Gain, DjFilter, CurveRate
  • DiscreteMap: Input, Scanner, Sync, RangeHigh, RangeLow
  • Indexed: IndexedA/B + borrows DiscreteMap group
  • Stochastic: Mask, GateLength, Tilt, Feel, Burst, Complexity, Contour, NoteDuration, Variation, Rest, Slide, Sleep, PatienceRhythm, Mutate, Jump

4 · Observations (the asymmetry)

Track types with nothing type-specific routable

  • PhaseFlux — rich per-stage nudges/warp shipped; zero in the enum (overhaul was waited on first). Only generic Track+Sequence route.
  • Teletype — no targets at all (Routing.cpp:284 bare break).
  • MidiCv — generic Track only.
  • Note — generic Track/Sequence only (the original homogeneous case).

Redundant / borrowed targets on older types

  • The flat Track block (Octave, Transpose, biases…) is offered to every type even where it's meaningless — older Note/Curve-era targets bolted onto heterogeneous engines.
  • Indexed borrows DiscreteMap's target group and owns IndexedA/B — dual-sourced, gated in two places.
  • Stochastic Divisor/Scale/RootNote special-cased because its divisor isn't a Routable like everyone else's.

Precedent Indexed already manages routing inside itself

Indexed doesn't use the global Target enum for its modulation. It carries a self-contained 2-slot mod-matrix in the sequence, with its own local vocabulary and its own config page:

IndexedSequence::RouteConfig { RouteSource source; // local source enum (not Routing::Source) ModTarget targetParam; // local target enum (Duration, …) float amount; // −200%..+200%, bipolar depth uint8_t targetGroups; // bitmask: which cell groups it hits } routeA, routeB + RouteCombineMode // 2 slots + combine → IndexedRouteConfigPage (its own UI)

This is a per-scope mod-matrix (Axis A, lower row) with assign-in-context UI — exactly the Bitwig shape, already in the codebase. It sidesteps all five global sources of truth. The open question is whether to generalize it.

5 · Proposed separation of concerns

Direction only — for discussion. Split routing into two tiers along the bucket boundary, each owning its own small source/target vocabulary instead of one 103-entry flat enum.

Tier 1 — Global mod matrix

Owns: the Whole-Performer bucket (Engine, Project, Bus).

Engine: one small central matrix, fixed slots source → global-target · depth. Sources = CvIn/Midi/Mod/Bus.

UI: one global routing page (today's RoutingPage, shrunk to global-only).

Tier 2 — Per-track mod matrix

Owns: per-track generic + per-track-type buckets.

Engine: each track carries its own slots (generalize Indexed RouteConfig). The track type declares its own routable targets in one place — kills the five-source smear; a missing leaf can't drift because the declaration is the leaf.

UI: per-track route page (Indexed pattern) and/or assign-from-parameter (long-press the param). Targets shown are exactly that type's — no inapplicable Octave on a Teletype track.

What this buys

  • One home per target. A track type lists its routable params once; no enum/targetInfos/dispatch/picker/leaf lockstep. PhaseFlux/Teletype get routing by declaring it locally, not by editing five global files.
  • No cross-type pollution. Generic Track targets stop being offered where meaningless; Indexed stops borrowing DiscreteMap.
  • Behavior vs engine split is clean: UI tier = "where do I make the route" (global page vs in-track / assign-from-param); engine tier = "who computes it" (central global matrix vs per-track matrices folded into the single-pass recompute the spine just built).
  • Bus stays the bridge between tiers (a track writes BusCvN; the global matrix or another track reads it) — the one explicit cross-tier patch point.

Open: fixed slot count per tier? bipolar depth + curve per slot (Indexed has amount only)? does the global modulator set (Mod1–8) stay global sources or become per-track? migration of existing routes off the flat enum (forward-compatible, no version bump).
