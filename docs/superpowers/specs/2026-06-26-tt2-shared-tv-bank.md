# TT Shared Variable Bank (`TV.A`–`TV.P`)

**Goal:** A bank of 16 named `int16` variables — `TV.A` … `TV.P` — **shared across all TT-family tracks** (full TT2 + Mini), so scripts on different TT tracks can pass values to each other. Like `BUS`, but: **ints not CV, TT-only (no routing), last-writer-wins (not summed)**.

- `TV.A` (read) pushes the slot's value; `TV.A 5` (set) writes it. Same get/set-by-position shape as the letter vars (`A`, `X`, …).
- **Shared:** one array on `Engine`; every TT track's `TV.*` ops hit the same 16 slots.
- **Ephemeral** = not serialized: `TV` values are never saved to or loaded from the project file. **Session-global** = zeroed once at `Engine` construction (boot), then holds its value across script runs, frames, transport stop, **and project load**, until overwritten or reboot. Never re-seeded per frame (unlike `BUS`). (`_tv` lives on `Engine`, outside `_model`, so a project load — which overwrites `_model` — doesn't touch it; per-project isolation would need new plumbing and is deferred.)
- **Last-writer-wins:** a plain store. If two tracks write `TV.A` in one frame, the last write stands (no accumulation).

> **Path convention:** `engine/…`/`model/…` are under `src/apps/sequencer/`; `teletype/…` are repo-root-relative.

## Why this mirrors existing mechanisms (the patterns to copy)

- **Letter vars are the op shape.** `A`/`B` are hand-written get/set ops (`engine/TeletypeNativeOps.cpp:34-58`); the rest are generated by `TT2_SIMPLE_VAR_OP(fn, member)` (`:87`) which expands to a get/set on `runtime.variables.<member>`. Each var is its own op (own `E_OP_*`, tokenizer rule, name, table slot). `TV.*` does the same — 16 ops via a sibling macro.
- **`BUS` is the engine-bank-via-host shape.** `BUS` reads `Engine::_busCv[4]` (`engine/Engine.h:135-160`) through `hostBusCv` (`TT2TrackEngine.cpp` → `_engine.busCv(index)`). `TV` mirrors this — an `Engine` array reached through host methods — **minus** the routing bridge that makes `BUS` system-wide (`CvRoute::InputSource/OutputDest::Bus`, `Routing.h:131`). `TV` adds NO `Routing` entries, so it stays TT-only.

## The store — `Engine`

Add to `Engine`:
```cpp
static constexpr int TvCount = 16;
int16_t _tv[TvCount] = {};                          // zero-init at construction; TT-only, not routed
int16_t tvGet(int i) const { return (i >= 0 && i < TvCount) ? _tv[i] : 0; }
void    tvSet(int i, int16_t v) { if (i >= 0 && i < TvCount) _tv[i] = v; }
```
- **Not re-seeded each frame** (BUS re-seeds `_busCvWriters` at `Engine.cpp:110`; `_tv` does not — it's a variable store). No per-frame code.
- **Lifetime = session-global.** Zero-init at construction (the member initializer above) and nowhere else. Do NOT clear on transport `reset()` (`Engine.cpp:875` — must persist through stop), and there is **no project-load clear** — `readProject` overwrites `_model` but `_tv` is on `Engine`, so it survives loads (`updateTrackSetups` only recreates an engine when `trackMode()` changes, `Engine.cpp:580`). This is the minimal, no-new-plumbing design that satisfies "ephemeral (not serialized) + persists." Per-project clear is Deferred.

## Host bridge — 2 methods

Add to `TT2Host` (`engine/TT2Host.h`, pure virtual) + override in **both** engines (no shared base — each implements the vtable):
```cpp
int16_t hostTvGet(uint8_t slot);
void    hostTvSet(uint8_t slot, int16_t v);
```
Both engine overrides are one-liners forwarding to the engine (mirroring `hostBusCv`):
```cpp
int16_t TT2TrackEngine::hostTvGet(uint8_t slot)            { return _engine.tvGet(slot); }
void    TT2TrackEngine::hostTvSet(uint8_t slot, int16_t v) { _engine.tvSet(slot, v); }
// identical bodies in TT2MiniTrackEngine.cpp
```
(2 methods × 5 sites: 1 `TT2Host.h` decl + 2 engine `.h` override decls + 2 engine `.cpp` defs.)

## The 16 ops — one macro

In `engine/TeletypeNativeOps.cpp`, a sibling of `TT2_SIMPLE_VAR_OP` that targets the shared bank via the host:
```cpp
#define TT2_TV_OP(fn, slot)                                                                       \
template<typename Cfg>                                                                             \
static void fn(TT2RuntimeT<Cfg> &, TT2OutputState &, const TeletypeProgramT<Cfg> *,                \
               int16_t *stack, uint8_t &stackSize, bool isSetPosition, TT2EvalError &error) {       \
    TT2Host *h = tt2ActiveHost();                                                                  \
    if (isSetPosition && stackSize >= 1) { int16_t v=0; if(!popStack(stack,stackSize,v,error)) return; if(h) h->hostTvSet(slot, v); } \
    else { pushStack(stack, stackSize, h ? h->hostTvGet(slot) : 0, error); }                       \
}
TT2_TV_OP(opTvA, 0)  TT2_TV_OP(opTvB, 1)  /* … */  TT2_TV_OP(opTvP, 15)
```
Null-guard `h` (null in editor/tests) exactly like the other host ops.

## Registration — through the teletype/ tooling (do NOT hand-edit generated files)

Same procedure as any op (source of truth = `match_token.rl`; `match_token.c`/`TT2OpNames.cpp` are generated):
1. **Enum** — add `E_OP_TV_A` … `E_OP_TV_P` (16) to `tele_op_idx_t` in `teletype/src/ops/op_enum.h`, before `E_OP__LENGTH`.
2. **Token rules** — add `"TV.A" => { MATCH_OP(E_OP_TV_A); };` … `"TV.P"` (16) to `teletype/src/match_token.rl`. (Confirm no `TV*` token exists today — grep; Ragel scanner is longest-match so no ordering hazard.)
3. **Regenerate** — `( cd teletype/src && ragel -C -G2 match_token.rl -o match_token.c )` (gitignored, per `PROJECT.md:407`); then `python3 teletype/utils/tt2_op_names.py` (regenerates `engine/TT2OpNames.cpp`). Never hand-edit the generated files.
4. **Vtable** — `table[E_OP_TV_A] = opTvA<Cfg>; …` (16) in the registration block of `engine/TeletypeNativeOps.cpp` (a small for-each macro is fine).

## Tests

- **Stub the 2 new pure virtuals in the existing fake hosts (compile-breaking otherwise).** `TT2Host`'s methods are pure virtual, and four tests define full fake hosts: `TestTeletypeV2Modulator.cpp` (`struct ModStubHost : TT2Host`), `TestTeletypeV2Geode.cpp`, `TestTeletypeV2EnvelopeOps.cpp`, `TestTeletypeV2LfoOps.cpp`. Add `int16_t hostTvGet(uint8_t) override { return 0; }` + `void hostTvSet(uint8_t, int16_t) override {}` to each (else they stay abstract and don't compile).
- **`TestTeletypeV2ParserContract.cpp`** — `expectToken("TV.A", E_OP_TV_A, …)` plus the boundary `TV.P` round-trips (catches a broken tokenizer entry).
- **`TestTT2OpNames.cpp`** — auto-passes once the names are regenerated (iterates `0..E_OP__LENGTH`).
- **Functional via a fake `TT2Host`** backed by a single `int16_t[16]` (the engines aren't host-constructible): `opTvA` set writes slot 0, `opTvA` get reads it back; **sharing** — two op invocations against the *same* fake host see each other's writes (a→write, b→read = same value); last-writer-wins (write 5 then 9 → reads 9); out-of-range slot is impossible (16 fixed ops), but `h==null` → get pushes 0. The engine-level "all tracks hit one `_tv`" is sim/manual (one `Engine`, two TT tracks).

## Budget

Negligible. `Engine._tv` = 32 bytes RAM. +16 op-table slots × 2 configs (~128 B CCMRAM) + 16 small op bodies × 2 instantiations (`.text`). No per-track/runtime growth.

## Deferred (propose only)
- **Per-project clear** — zero `_tv` on project load/clear (so values don't carry between projects). Needs a new `Engine` load hook (`readProject`/`resume` have none today); v1 is session-global.
- Persist `TV.*` with the project (the opposite — drop "ephemeral") — add serialization; not in v1.
- An indexed `TV i` alias alongside the named slots — only if scripts want computed indices.
