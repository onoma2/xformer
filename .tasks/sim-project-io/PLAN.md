# Plan: Simulator Project I/O

## Goal
Give the `testsim` Python module the ability to read/write hardware projects through the same code path the hardware uses (FileManager → FatFs → SdCard → sdcard.iso), with full two-way format compatibility.

## Steps (dependency order)

### Step 1: Generate `sdcard.iso` with FAT filesystem
The simulator's `SdCard.h` reads/writes a 512KB disk image (`sdcard.iso`, 1024 sectors x 512 bytes). Without it, the virtual SD is a zeroed buffer and FatFs can't mount.

**Approach A — script (preferred):**
```bash
mkfs.fat -F 12 -s 1 -S 512 -C sdcard.iso 1024
# Mount and create directory structure
hdiutil attach sdcard.iso -mountpoint /tmp/sd
mkdir /tmp/sd/PROJECTS /tmp/sd/SCALES /tmp/sd/TELS /tmp/sd/TELT
hdiutil detach /tmp/sd
```

**Approach B — runtime formatting:**
In `SequencerApp` constructor or boot, detect unformatted volume and call `f_mkfs()` + `f_mkdir()`.

**Files:** `src/platform/sim/drivers/SdCard.h`, potentially `SequencerApp.h`

### Step 2: Export FileManager to Python
Add pybind11 bindings for `FileManager` static methods in `src/apps/sequencer/python/`:

```cpp
py::class_<FileManager> fileManager(m, "FileManager");
fileManager
    .def_static("volumeMounted", &FileManager::volumeMounted)
    .def_static("volumeAvailable", &FileManager::volumeAvailable)
    .def_static("format", &FileManager::format)
    .def_static("writeProject", &FileManager::writeProject, py::arg("project"), py::arg("slot"))
    .def_static("readProject", &FileManager::readProject, py::arg("project"), py::arg("slot"))
    .def_static("slotInfo", &FileManager::slotInfo, py::arg("type"), py::arg("slot"))
    .def_static("slotUsed", &FileManager::slotUsed, py::arg("type"), py::arg("slot"))
;
```

**File:** `src/apps/sequencer/python/sequencer.cpp` (or new `filemanager.cpp`)

### Step 3: Fix existing `loadProject()` hash validation
The Python binding's `loadProject` in `project.cpp` doesn't call `reader.checkHash()`, unlike `FileManager::readProject` which returns `INVALID_CHECKSUM` on mismatch.

**Fix:** After `project.read(reader)`, call `reader.checkHash()` and throw on failure.

**File:** `src/apps/sequencer/python/project.cpp`

### Step 4: Test round-trip
Write a Python test script that:
1. Creates `testsim.Environment()`
2. Calls `env.simulator.wait(10)` to boot firmware + initialize FS
3. Checks `FileManager.volumeMounted()`
4. Creates a project, sets some values, calls `FileManager.writeProject(project, 0)`
5. Loads slot 0 back, compares values
6. Verifies the `.PRO` binary written to virtual SD matches hardware format

**Files:** `src/apps/sequencer/tests/test_project_io.py`

### Step 5: sdcard.iso placement
Decide where the `sdcard.iso` lives at runtime:
- In the repo root (tracked by git)
- Generated by `scripts/` during build/CI
- Created on first simulator run if missing
