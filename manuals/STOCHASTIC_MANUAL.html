1 · Overview

The Stochastic track is a probability-driven generative sequencer. Instead of programming every note, you define a probability space and the sequencer explores it — generating rhythm and melody from high-level parameters in real time.

Figure 1: Full generative pipeline — every parameter rolls dice per step, competing to shape the output. Loop permutations accumulate until evolution (mutate/patience) alters the buffer.

The track operates on two independent generative domains — rhythm (when events fire) and melody (what pitches they play). Each domain has its own source mode, seed, buffer, and mutation system. The two outputs combine at the gate and CV jacks.

Tip: The Stochastic track excels at generative ambient, aleatoric percussion, evolving basslines, and any situation where you want musical variation without step-by-step programming.

3 · Key Concepts

3.1 · Loop vs Live Source Modes

Figure 3: Loop mode generates a buffer then mutates it; Live mode generates each step fresh

Each domain (rhythm, melody) has an independent source mode. Loop fills a pattern buffer on first play and can evolve via Mutate/Patience. Live generates every step fresh — maximum unpredictability.

3.2 · Degree Tickets & Weighted Pitch Selection

Figure 4: Ticket weights determine pitch probability. Higher bars = more likely to be selected.

Each degree of the current scale gets a ticket count (0–100). More tickets = higher probability. A value of -1 excludes that degree entirely.

3.3 · Complexity, Contour & Linearity

Figure 5: Visual behavior of the three pitch modifiers — how they reshape weight distributions

3.4 · Mutation & Patience (Evolution System)

Figure 6: Evolution timeline — mutation makes single-step changes, patience triggers full resets

Mutate replaces one step per cycle. Patience controls how many cycles before boredom triggers a full buffer regeneration. When patience is low, patterns reset frequently. When high, patterns persist with only gradual mutation.

3.5 · Burst System

Figure 7: A parent note generates 1–4 child sub-notes within its duration

Any step may generate children — sub-notes that fire inside the parent duration. Burst Count controls how many children, Burst Rate controls their spacing. Burst Pitch chooses whether children inherit the parent pitch or generate their own.

3.6 · Density Ranking & Tilt

Figure 8: Density gate with Tilt biasing rhythm toward early steps

3.7 · Jump Register & Lock

Jump probability fires at cycle end, toggling an octave register between -1, 0, and +1 — creating sudden transposition shifts. Lock freezes the entire evaluated pattern (all pitches, gates, children) so it plays back identically every cycle.

9 · Workflow Guide

9.1 · Quick Start

  1. Create a track, set mode to Stochastic.
  2. Open STOCH. Set Rhythm + Melody to Loop.
  3. Set Density to 80, Rest to 10.
  4. Set Complexity to 40.
  5. Open Tickets, give root + fifth high weights, others low.
  6. Press play. The pattern repeats with subtle variation.

9.2 · Slow Mutation

9.5 · Lock a Happy Accident

When a pattern sounds great: open STOCH CFG and set Lock to On. Every pitch, gate, burst, and slide freezes exactly as-is. Unlock to resume evolution.

9.6 · Polyrhythmic Phasing

Create two Stochastic tracks with similar settings. On the second, set Rotate to +3. The second track reads its buffer shifted by 3 steps, creating polyrhythmic interplay.

9.7 · External CV Routing

  • Route an LFO to Density for pulsing intensity.
  • Route a sequencer lane to Contour for melodic arcs.
  • Route random voltage to Mutate for evolution bursts.
  • Route an envelope to Slide Prob for portamento swells.
