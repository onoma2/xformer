1 · Overview

The Stochastic track is a probability-driven generative sequencer. Instead of programming every note, you define a probability space and the sequencer explores it — generating rhythm and melody from high-level parameters in real time.

The track operates on two independent generative domains — rhythm (when events fire) and melody (what pitches they play). Each domain has its own source mode, seed, buffer, and mutation system. The two outputs combine at the gate and CV jacks.

Sequence Ownership: Generative parameters (all per-event knobs, ticket weights, Loop window, mutation rates) live on the Sequence (Pattern), so every pattern in a Stochastic track can carry its own probability shape and evolution rate. Track-level settings (Lock, Divisor, ClockMult, Scale, RootNote, Octave, Transpose, SlideTime, FillMode) live on the Track and survive pattern switches.

3 · Key Concepts

3.1 · Loop vs Live Source Modes

Figure 3: Loop mode generates a buffer then mutates it; Live mode generates each step fresh

Each domain (rhythm, melody) has an independent source mode. Loop fills a pattern buffer on first play and can evolve via Mutate/Patience. Live generates every step fresh — maximum unpredictability.

3.2 · Degree Tickets & Weighted Pitch Selection

Figure 4: Ticket weights determine pitch probability. Higher bars = more likely to be selected.

Each degree of the current scale gets a ticket count (0–100). More tickets = higher probability. A value of -1 excludes that degree entirely.

3.3 · Complexity, Contour & Repeat

Three knobs reshape the per-step pitch probability after the degree tickets are weighed:

  • Complexity — triangle kernel around the last played degree. Low values keep the next pick close to where you just were (tight melodic focus); high values let it leap freely across the scale. Width is symmetric around the last degree and the kernel is multiplied 10× against the leakage floor so the center always wins at small knob values (no integer-truncation cliff).
  • Contour (-100–+100) — directional drift. Each fired step nudges the kernel's center by a small offset along the contour direction; positive Contour walks upward, negative walks downward. Drift is capped to the kernel width so it cannot run away in large scales.
  • Repeat (0–100) — Bernoulli per-event chance to bypass generation entirely and replay the last emitted event verbatim. Works in both Loop and Live mode. Freezes/clusters at the playback layer. Lives in the _linearity model slot for serialization stability.

Application order: tickets × Complexity kernel × Marbles bell × Steps sieve; Contour drifts the kernel's center between picks; Repeat is a separate bypass.

3.4 · Mutation & Patience (Evolution System)

Figure 6: Evolution timeline — mutation makes single-step changes, patience triggers full resets

Mutate is bipolar. Positive values permute one event per cycle within the active [first, last] window (Marbles-style swap). Negative values destructively regenerate one event per cycle, treating the existing Loop material as raw clay. Zero holds the buffer.

Patience is split per domain: Patience R independently counts loop cycles for the rhythm domain, Patience M for melody. Each rolls a Poisson-CDF probability per cycle; on success, that domain's loop tape is invalidated and regenerated on the next trigger. Knob 0–100 maps to λ ∈ [1, 80] linear — no off sentinel. Track Lock is the proper "freeze" gesture.

Ordering matters: at cycle end, Patience rolls before Mutate. If Patience invalidates a domain, Mutate skips that domain (no wasted writes that audio never observes).

3.5 · Burst System

Figure 7: A parent note generates 1–4 child sub-notes within its duration

Any step may generate children — sub-notes that fire inside the parent duration. Burst Count controls how many children, Burst Rate controls their spacing. Burst Pitch chooses whether children inherit the parent pitch or generate their own.

3.6 · Mask, Tilt & Steps Sieve

Three deterministic filters thin or shape the Loop tape after generation. All three operate on the active playback window [first, last] only — off-window events are never read by playback, so they don't get "top priority" slots in the rank.

  • Mask (0–100) — ranked thinning. Each event in the window receives a rank 0..windowSize-1. The mask passes events whose rank × 100 < mask × windowSize. Knob 100 = everything plays; knob 0 = nothing plays.
  • Tilt (-100–+100) — duration-aware rank order. Positive Tilt favors long notes (long durations get low ranks → survive Mask cuts). Negative Tilt favors short notes. Zero is random rank.
  • Steps (0–100) — pitch sieve cutoff. Acts on the degree picker upstream: only the top K of N degrees (ranked by ticket weight) are eligible. Steps = 100 means all degrees survive (no sieve); Steps = 0 collapses to a single degree (the highest-ranked one).

Note: the legacy Density knob has been repurposed as Gate Length spread (see 3.9). The current sieve role lives on Steps; the current rank-thinner role lives on Mask.

3.7 · Jump Register & Lock

Jump probability fires at the end of each pattern cycle (Bernoulli against the knob). On success, an internal jump register walks ±1 octave (Proteus-style), reflected at ±2 octaves. The register is applied as a playback transposition only — stored events are never modified, so turning Jump off snaps the sequence back to its captured pitches.

Lock freezes the evaluated pattern (all pitches, gates, children, slide flags) so it plays back identically every cycle. Useful for capturing a "happy accident". Lock is a track-level setting on the Config page.

3.8 · Marbles Bell & Steps Sieve (Pitch Picker)

The pitch picker is a four-stage product applied per non-rest event:

  1. Degree tickets — per-degree base weight (0–100), edited on the Pitch page.
  2. Complexity kernel — triangle around the last picked degree, width modulated by knob.
  3. Marbles bell — gaussian envelope across scale degrees centered on Bias (0–100 = low-to-high) with width Spread (0–100). Tight Spread = fixed register; wide Spread = full scale.
  4. Steps sieve — rank cutoff that keeps only the top K of N degrees by post-product weight. K = ceil(N × steps / 100).

Whichever degree survives all four stages is emitted; if Steps collapses to a single degree, the picker becomes deterministic for that pool.

3.9 · Gate Length Spread

Gate Length (0–100, formerly the Density slot) is a triangle distribution width for per-event gate length. Knob 0 = exactly 50% of the duration. Higher values spread the picked gate length symmetrically around 50% (still floored to a minimum audible width). Each event picks its own gate length at trigger time. Lives on the LIVE page at step 13.

6 · Ticket Pages

Two ticket editors live under EDIT, paged via F5 NEXT: the Pitch ticket page (per-degree weights for the current scale) and the Duration ticket page (per-slot weights for the 8-slot duration LUT).

6.1 · Pitch Tickets

Bar graph of per-degree weights for the current scale (up to 16 degrees per bank; larger scales use left/right banks). The selected degree is highlighted with an underline; the held / focused degree value reads out top-left in Small font. DROT/MROT rotation values display top-right in Tiny font.

Footer FnAction

Quick-edit (Page + step on ticket pages):

  • Page + step 1 — INIT (all tickets = 100, rotations = 0)
  • Page + step 2 — EVEN (copy current degree's value to all)
  • Page + step 3 — RAND (random tickets 0–100)

6.2 · Duration Tickets

8-slot duration weight editor. Slot index corresponds to the LUT entry that drives noteDuration picks (slot 0 = longest, slot 7 = shortest, slot 5 = ×1 = divisor). Each slot's ticket weight multiplies into the Variation kernel at generation time, so duration tickets are the "tonal" equivalent of pitch tickets for rhythm. Slot labels are generated at runtime from the active divisor (e.g. "1/8" at divisor=1/8).

The top-right corner of the page shows the current Rest % in Tiny font, always on (the live rest probability never disappears, even when focus is on a duration slot). When focus is switched to Rest via F3, the same value also appears top-left in Small font as the primary readout.

F2 — NewRRegenerate the rhythm loop tape. Double-click guarded — first press shows "Press again to renew rhythm"; second press fires and toasts "Rhythm renewed".
F3 — RSTSwitch encoder focus to Rest probability. Toggle off (back to duration ticket) by pressing again.

8 · Generator Engine Deep Dive

8.1 · Duration Mapping

Figure 11: Duration LUT picker. Variation spreads a triangle kernel around the Bias slot.

Each LUT slot is a duration fraction of the active clock divisor — not a fixed musical value. The slot labels you see in the Duration ticket editor (e.g. "1/8") are computed at runtime from the active divisor, so changing the divisor automatically retunes the LUT. Per-slot ticket weights (Duration page) multiply into the Variation kernel: a slot with weight 100 is fully eligible; weight 0 is muted from the pick.

9 · Workflow Guide

9.1 · Quick Start

  1. Create a track, set mode to Stochastic.
  2. You start on the LIVE page (default for Stochastic tracks).
  3. The footer Fn keys F1/F2 toggle Loop/Live per domain. Leave both on Loop for now.
  4. On LIVE, hold step 2 (R = Rest) and turn the encoder to a small value (10–20). Multi-hold several step buttons to edit them at once; the overlay reads MANY SELECTED.
  5. Press F5 to NEXT through to PITCH. Press EDIT to enter ticket editing if you want to weight specific scale degrees.
  6. Hit play. The Loop tape generates on the first cycle; subsequent cycles replay it with whatever Patience / Mutate / Sleep evolution you've set.

9.5 · Lock a Happy Accident

When the pattern lands somewhere good: CONFIG → set Lock to On. Every pitch, gate, slide, and burst child freezes — further Patience / Mutate rolls are still evaluated but the engine plays the cached evaluated buffer instead. Unlock to resume evolution. (Note: Lock is currently a runtime cache; its persistence is under review — see docs/stoch-review.md.)

9.6 · Polyrhythmic Phasing

Create two Stochastic tracks with similar settings. On the second, set Rotate to +3 on the LOOP page. The second track reads its buffer shifted by 3 steps within the active [first, last] window, creating polyrhythmic interplay.

9.7 · External CV Routing

  • Route an LFO to Mask for pulsing density.
  • Route a sequencer lane to Contour for melodic arcs.
  • Route random voltage to Mutate for evolution bursts.
  • Route an envelope to Slide for portamento swells.
  • Route a slow LFO to Bias for register sweeps under the Marbles bell.
