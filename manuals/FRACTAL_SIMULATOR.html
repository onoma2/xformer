Ornament rate sets how often an ornament fires (P per gated cell); intensity sets how fancy: low = 2-step flourishes only, mid adds 4-step, high adds the 8-step trill. Decoupled, so you can have elaborate-but-occasional. When one fires, a random ornament from the eligible tier plays as extra short timbre-B notes around the main note — the trunk is never modified. The main note plays raw (literal trunk CV); ornament notes snap to scale degrees of the inherited Scale + Root around it. "Toward/Away" steps use the next trunk cell. Ornaments fire only when the sounding trunk cell is inside the ornament zone (pink O◀ O▶ handles on the trunk strip); outside it the main note plays plain.

Fractal — Branches (KD-12)

—
16
—
The played phrase is Trunk → B1 → … → BN, looped (N = branch count, 0–7). Each segment is one loop-window length, so total length = loopLen × (1 + N). Each branch is a chained transform of the previous segment (B1 transforms the trunk): Transpose / Reverse / Inverse / Ret-Inv / Rotate / Compress / Expand / Gate-thin, drawn deterministically from the seed but only from the ops enabled in the transform pool (empty pool → all Transpose). Reseed rerolls; editing the trunk also reseeds. Path is an N-bit word setting the branch navigation order — bit (b-1) places branch b in the outward leg (0) or holds it for the return leg (1). The route is Trunk → outward branches (ascending) → held branches (descending): all-0 = Forward Ladder, all-1 = Reverse Ladder. Every segment plays exactly once, so total length is unchanged. Branches are read-only transforms of the trunk — never written to it.

Gate-state glyphs

· rest (gateLen 0, no trig)
▮ trig (gateLen 1, short)
▤ proportional (2–14, gate fills part of section)
═ tie (15, full section / legato)
∅ empty (invalid — outputs root, gate off)
Trunk cell = {cv, gateLen, valid}. gateLen 0=rest, 1=trig, 2–14=proportional, 15=tie, mirroring the 4-bit field in the hardware spec.
