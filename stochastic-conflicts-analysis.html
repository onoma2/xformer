Stochastic Engine Analysis: Conflicts & Knob Laws

This table outlines DSP constraints, edge-case behaviors, and the underlying mathematical mapping (Linear vs. Exponential) of the parameter knobs to evaluate their musical feel.

RANG
(Pitch Range)		 Set to 0
Extreme Behavior		 Octave Jumps. In the bipolar-Range model, Range=0 does NOT kill pitch shaping. It produces a single-octave field with a 100% per-slot octave-displacement chance. Pitch generation continues, heavily biased toward jumps. CMPX, CONT, BIAS, and SPRE still fully operate. Linear

(Spread)		 Set to 0

(Repeat Prob)		 Set to 100%
Dead Zone		 Freezes Live Generation. If the real-time engine is repeating the historical buffer at 100%, new values are never generated; it continuously replays _lastStepContent. Linear Probability
(0-100%)		 Keep Current

Loop Page & Window Interactions

Parameter(s) Interaction / Issue Behavior & Explanation Current Law Proposed Law
Patience
+ Mutate		 Patience locked (100)
Independence		 Mutate Continues. Patience invalidates loop sources and triggers full regeneration. Mutate is an independent cycle-end hook. Setting Patience to 100 ("no regen ever") does NOT block Mutation; Mutate still fires. Linear Lambda
(1.0 + p*0.79 mapping 1 to ~80 loops)		 Change to Exponential
A linear scale makes the top half of the knob (40-80 loops) mostly imperceptible. An exponential curve would provide finer resolution for short waits (1-8 loops) while compressing the extreme wait times.
